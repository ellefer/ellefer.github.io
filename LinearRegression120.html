The first thing we are going to do is perform linear regression in R. Let's start with the salary data. Can we explain a salary by the number of years experience? The command lm performs the linear regression. Salary ~ YearsExperience reads "How is Salary explained by Years Experience?". Salary is the dependent variable. (This is important to note because usually we put the x variable first when we plot data).

reg &lt;- is saving the results of linear regression to the variable reg

reg<-lm(formula = Salary ~ YearsExperience,data=S)
Messages

Let's print out the results of the linear regression using the command summary.

From this "summary" (a long summary!) we can get the \textbf{equation of the line}: the intercept is listed right after (Intercept) - 25792.225792.225792.2, the slope is listed right after the independent variable - 9450.09450.09450.0. We can interpret this-- our line predicts that if you have no experience, your salary will be 25792.2 dollars (this is only a valid prediction if it is within our data), and that for every year of experience you gain, your salary increases by 9450.0 dollars (if only!). Our line : y^=25792.2+9450x,\hat{y} = 25792.2+9450x,y^â€‹=25792.2+9450x, where xxx is years experience.

We can also get the r2r^2r2 value, listed right after the multiple r^2: .957. This means that 95.7% of the variation in salary is explained by the variation in experience. A good fit!

To get the rrr value- take the square root of r2r^2r2, and add the sign of the slope. Our slope is positive, so our rrr value is .957â‰ˆ.978\sqrt{.957} \approx .978.957â€‹â‰ˆ.978 - a strong, positive relationship between experience and salary.

Finally, we talked about residual standard error (RSE) -- this helps us identify outliers. It is the standard deviation of the residuals. So, if a residual for an individual data point is more than 2âˆ—RSE2RSE2âˆ—RSE away from the line, we can flag this as an outlier. Our RSE is 5788, so if the residual for an individual data point is more than 2âˆ—5788257882âˆ—5788, we should investigate this as an outlier.

We always should plot the data! Now we are going to plot the data with the regression line on top of it. We first can grab the slope and the intercept from the saved regression information in the variable reg. If you repeat this for your own data and call your regression information reg, you shouldn't have to change the code in the following cell outside of inputting your own RSE value. 

coeff<-coefficients(reg)   #gets the slope and the intercept and stores it in coeff       
intercept<-coeff[1] #stores the intercept in variable intercept
slope<- coeff[2] #gets the slope and stores it in the variable slope
RSE = 5788 #storing SRE from above
Messages

Now, we plot the regression line along with the data. The first line ggplot, sets up the axes. We want our independent variable to be YearsExperience and our dependent variable to be Salary (this is the opposite order we input them when we called lm).

geom_point plots the scatter plot of the data

The first two geom_abline plots the regression line, where we input the slope and the intercept we saved from before. We choose to plot the regression line in red.

The second two geom_ablines plots the bounds for outliers. These lie 2*RSE away from the regression line. If a data point is outside these lines, its error is two standard deviations away from the regression line.

ggplot(S, aes(YearsExperience, Salary)) +
  geom_point()+ 
geom_abline(intercept = intercept, slope = slope, color="red", 
               linetype="dashed", size=1.5) + #plots the regression line(you shouldn't have to change this line)
geom_abline(intercept = intercept - 2*RSE, slope = slope, color="yellow", 
               linetype="dashed", size=1.5)+ #plots the bound for outliers (you should only have to change the 5788 to your RSE value)
geom_abline(intercept = intercept + 2*RSE, slope = slope, color="yellow", 
               linetype="dashed", size=1.5)+ #plots the bound for outliers (you should only have to change the 5788 to your RSE value)
  ggtitle("Years Experience & Salary Regression")    #changes the title
